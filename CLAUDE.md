# Scaffold Canton

Canton Network dApp scaffold with Vault contract, tRPC backend, React frontend, and wallet gateway.

## Example vs. Your Code

The Vault contract (`daml/daml/Vault.daml`), its backend domain logic (`backend/src/domain/vault.ts`), generated template IDs, router endpoints (`user.createDepositRequest`, `user.acceptDeposit`, etc.), and the frontend `VaultTab.tsx` are **examples**. They demonstrate the full pattern: Daml contract → codegen → backend commands/PQS queries → frontend (wallet gateway writes + tRPC reads).

When a user forks this repo to build their own dApp, follow this order:

1. **Create contracts** — write `.daml` files in `daml/daml/`
2. **Build DARs** — `cd daml && daml build`
3. **Run codegen** — `scripts/daml-codegen.sh -i daml/.daml/dist -o backend/src/generated`
4. **Create backend logic** — domain builders, PQS queries, tRPC routes (use `domain/vault.ts` + `router/user.ts` as reference)
5. **Create frontend logic** — tabs importing `TEMPLATE_IDS` from generated code (use `VaultTab.tsx` as reference)

Ask the user if they want to keep or remove the Vault examples.

## Services

- `backend/` — Hono + tRPC server (:8080)
- `frontend/` — React + Vite dashboard (:5173)
- `wallet-gateway/` — Splice Wallet Kernel (:3030)
- `daml/` — Vault smart contract (Daml 3.4)

## Start

```bash
cd backend && npm run dev &
cd frontend && npm run dev &
cd wallet-gateway && npm run start &
```

## Key Patterns

- **Ledger API template IDs need full package hash** (Canton 3.4.12+): `7c39...:Vault:DepositRequest`
- **PQS template IDs use no hash**: `Vault:DepositRequest`
- Hashes generated by `scripts/daml-codegen.sh` into `backend/src/generated/template-ids.ts`
- **Wallet writes**: dApp SDK `prepareExecute` → wallet gateway signs
- **Admin writes**: tRPC → backend → Ledger API (Keycloak JWT)
- **All reads**: tRPC → backend → PQS (public endpoints, no JWT for vault)
- PQS must be re-indexed after uploading new DARs

## Config

- `backend/.env.local` — Ledger API, PQS, Keycloak, validator URLs
- `frontend/.env.local` — OIDC, validator, ledger, wallet gateway URLs
- `wallet-gateway/config.json` — Keycloak IDP, network, signing store (gitignored)
